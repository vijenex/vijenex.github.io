How to Contribute

Learn how to write blog posts, contribute code, and share feedback with the Vijenex™ community.

Writing Blog Posts

Vijenex™ blog posts are file-based and managed through Git. There is no web-based editor or CMS. All content is written as static HTML or Markdown files.

Step 1: Choose Your Format

You can write blog posts in either format:

  • HTML: Use the template at templates-static/blog-post.html
  • Markdown: Write in Markdown, then convert to HTML using the Markdown to HTML tool

Step 2: Create Your Blog Post

Follow this folder structure:

blog/
  posts/
    YYYY/
      MM/
        your-post-slug/
          index.html
          meta.json

Example: For a post published in November 2025 with slug "my-security-tips":

blog/posts/2025/11/my-security-tips/index.html
blog/posts/2025/11/my-security-tips/meta.json

Step 3: Create the HTML File

Copy the template from templates-static/blog-post.html and fill in:

  • Title (replace BLOG_TITLE)
  • Content (write your blog post)
  • Meta tags (SEO information)
  • Author information

Step 4: Create the meta.json File

Create a meta.json file with this structure:

{
  "id": "your-post-slug",
  "title": "Your Blog Post Title",
  "date": "2025-11-28",
  "category": "Security",
  "excerpt": "A brief description of your post.",
  "author": "Your Name",
  "readTime": "5 min read",
  "url": "posts/2025/11/your-post-slug/index.html",
  "published": true
}

Step 5: Add Images (Optional)

Place images in:

blog/posts/2025/11/your-post-slug/images/

Reference them in your HTML as:

<img src="images/your-image.png" alt="Description">

Step 6: Update posts.json

Add your post metadata to blog/_data/posts.json:

[
  {
    "id": "your-post-slug",
    "title": "Your Blog Post Title",
    "date": "2025-11-28",
    "category": "Security",
    "excerpt": "A brief description.",
    "author": "Your Name",
    "readTime": "5 min read",
    "url": "posts/2025/11/your-post-slug/index.html",
    "published": true
  }
]

Step 7: Submit via GitHub

  1. Fork the vijenex.github.io repository
  2. Create a new branch: git checkout -b blog/your-post-slug
  3. Add your files: git add blog/posts/2025/11/your-post-slug/
  4. Commit: git commit -m "Add blog post: Your Title"
  5. Push: git push origin blog/your-post-slug
  6. Create a Pull Request on GitHub

Step 8: Review Process

After submitting your PR:

  • The Vijenex team will review your post
  • We may suggest edits or improvements
  • Once approved, your post will be merged and published

Contributing Code

To contribute code to Vijenex™ projects:

  1. Browse our GitHub repositories
  2. Fork the repository you want to contribute to
  3. Create a feature branch
  4. Make your changes
  5. Submit a Pull Request

Sharing Feedback

Have feedback, bug reports, or feature requests?

  • For website issues: Open an issue on the website repository
  • For tool issues: Open an issue on the specific tool repository

Questions?

If you have questions about contributing, please open an issue or reach out through GitHub.
